## 2. 资料已确认的事实

### C-01 现行高中课程范围包含圆锥曲线方程

优先级：P0  
阻塞：M1.5 范围冻结  
状态：资料已确认

截至本次核实，教育部公开的现行普通高中数学课程依据仍是《普通高中数学课程标准（2017 年版 2020 年修订）》。其中“圆锥曲线与方程”明确包含：

* 椭圆的定义、标准方程和简单几何性质；
* 抛物线、双曲线的定义、几何图形、标准方程和简单几何性质；
* 椭圆、抛物线的简单应用；
* 通过曲线与方程进一步体会数形结合。

该内容位于选择性必修的平面解析几何主题。若产品面向参加高考的高中用户，圆锥曲线不应放到遥远的“可能扩展”中。

圆在课标中以“圆与方程”单列，要求标准方程、一般方程及基本位置关系；产品应支持圆，但 UI 术语宜与课标一致。

来源：

* [教育部：关于印发普通高中课程方案和课程标准（2017 年版 2020 年修订）的通知](https://www.moe.gov.cn/srcsite/A26/s8001/202006/t20200603_462199.html)
* [人民教育出版社：普通高中数学课程标准（2017 年版 2020 年修订）PDF](https://www.pep.com.cn/xw/zt/rjwy/gzkb2020/202205/P020220517519489596282.pdf)
* [人民教育出版社：选择性必修第一册圆锥曲线互动资源](https://www.pep.com.cn/xw/zt/xkzt/gsazy/gsaxbx1/)

剩余风险：教育部曾公开提及继续修订完善高中课程方案和课程标准；1.0 发布候选前应再次检查是否已有新版数学课标正式发布。

### C-03 Windows 10 已结束常规支持，但 Qt 6.11 仍有兼容范围

优先级：P0  
阻塞：M0 依赖基线、M6 发布声明  
状态：资料已确认，待实机与产品决策

Microsoft 已于 2025-10-14 结束 Windows 10 常规支持。系统仍可运行，但不再默认获得常规技术支持和安全更新。另一方面，Qt 6.11 的官方平台表仍列出 Windows 10 1809+ x86_64 和 Windows 11，且注明 Qt 6.12 将是最后支持 Windows 10 的 Qt 版本。

因此必须区分：

* “Qt 技术上仍支持某 Windows 10 配置”；
* “Microsoft 仍对该操作系统提供常规安全支持”。

PRD 暂定 Windows 11 为主要支持，Windows 10 作为课堂存量设备兼容目标，不能把 Windows 10 写成与 Windows 11 同等级、无风险的长期平台。

来源：

* [Microsoft：Windows 10 support has ended](https://support.microsoft.com/en-US/Windows/Deployment/Updates-Lifecycle/windows-10-support-has-ended-on-october-14-2025)
* [Qt 6.11：Supported Platforms](https://doc.qt.io/qt-6/supported-platforms.html)

审计补充（2026-07-19）：当前开发机经 Microsoft OS Build 对照确认为 Windows 11 25H2（Build 26200.8655），而不是 Windows 10。用户确认该机此前已完成最小 `y=x^2` 绘图冒烟，并提供既有 `deployment/main.dist/main.exe` 打包目录截图；阶段 0A 还完成了 `main`、`main_window`、`plot_engine` 的非交互导入检查。上述证据仅说明当前 Windows 11 开发机上的早期演示可最小运行/已有打包产物，不构成 Windows 10 实机验证、完整 GUI 冒烟、正式发布验证或 P0-01 关闭依据。

### C-04 当前 Qt for Python 的 Python 与线程基础要求

优先级：P0  
阻塞：M0  
状态：资料已确认，待锁定版本和原型

当前 Qt for Python 入门文档要求官方 Python 3.10+。项目本地 `pyproject.toml` 目前要求 Python `>=3.12`，并使用 PySide6 `>=6.11.1`；发布前仍需把开放下限改成经过测试的可复现版本组合，而不是无限制追随新版本。

Qt 官方推荐的 worker-object 模式包括：创建 `QObject` Worker、用 `moveToThread()` 移动、通过 queued signal/slot 通信、在线程结束时 `deleteLater()`，关闭时 `quit()`/`wait()`。`QThread` 对象本身仍生活在创建它的线程；强制 `terminate()` 被明确视为危险做法。

来源：

* [Qt for Python：Getting Started](https://doc.qt.io/qtforpython-6/gettingstarted.html)
* [Qt for Python：QThread](https://doc.qt.io/qtforpython-6/PySide6/QtCore/QThread.html)
* [Qt for Python：Threads and QObjects](https://doc.qt.io/qtforpython-6/overviews/qtdoc-threads-qobject.html)

### C-05 Qt 6 的高 DPI 基线

优先级：P1  
阻塞：M5  
状态：资料已确认，待实机

Qt 6 在 Windows 上默认使用 Per-Monitor DPI Aware V2，并以设备无关像素处理大多数 Widget 几何；QImage、QPixmap 等像素缓冲仍需要正确处理 device pixel ratio。官方文档同时指出，非整数缩放可能暴露视觉问题，因此 125%、150%、175% 等比例必须实测。

来源：

* [Qt for Python：High DPI](https://doc.qt.io/qtforpython-6/overviews/qtdoc-highdpi.html)

### C-06 Windows 截图协议已变化

优先级：P0  
阻塞：M4  
状态：资料已确认，待打包与原型决策

Microsoft 当前文档使用新的 `ms-screenclip` 截图协议，并说明：通过 `redirect-uri` 安全接收结果需要具有应用身份的已打包 Windows 应用；未打包 Win32 调用者不能通过该回调取得结果。旧的截图启动文档已标记为弃用。

这意味着原 PRD 的“调用系统截图工具 → 结果一定进入剪贴板 → 监听一次变化”不能作为稳定契约。M4 必须在以下方案间用原型选择：已打包回调、用户主动粘贴，或经证明可靠的短时剪贴板观察。

来源：

### C-09 Matplotlib 线程与非交互式渲染边界

优先级：P0  
阻塞：M1 RenderActor、M1.5  
状态：资料已确认，待本项目原型

Matplotlib 官方说明其整体不是线程安全的；多线程使用时应由应用串行化 Artist 访问，并使用非交互式后端。这支持项目采用唯一 RenderActor、Agg、Figure/Canvas 局部所有权和有序资源释放。

`contour` 能绘制二维网格的零等值线是库能力事实，但“能绘制”不等于它是本项目已分类直线/圆锥曲线的最佳主方案。当前架构决策改用几何参数化采样；contour 只保留为诊断对照或未来任意隐式曲线的独立候选，不据此扩张 PRD 范围。

来源：

* [Matplotlib FAQ：Work with threads](https://matplotlib.org/stable/users/faq.html#work-with-threads)
* [Matplotlib：contour API（能力事实与诊断对照）](https://matplotlib.org/stable/api/_as_gen/matplotlib.pyplot.contour.html)

仍需原型确认：固定版本下 FigureCanvasAgg 的字体/资源初始化与关闭是否稳定，RenderActor latest-wins/取消/关闭是否泄漏 Figure；参数化直线/圆锥曲线的可见参数区间、分支、裁切、性能、内存与残差容差也不能只凭文档决定。

### C-11 未成年人和课堂截图具有较高隐私风险

优先级：P0  
阻塞：M3 真实 OCR、M6 发布  
状态：法律事实已确认，具体方案待法务

《中华人民共和国个人信息保护法》将不满十四周岁未成年人的个人信息列为敏感个人信息，并对处理目的、必要性、严格保护措施、单独同意、监护人同意和专门规则提出要求。课堂截图可能同时包含姓名、成绩、头像或其他学生信息，不能仅靠一句“请谨慎上传”完成合规设计。
